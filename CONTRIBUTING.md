# 参与贡献

感谢你改进 `mp-click-to-source`。这个项目只处理微信小程序开发期的源码定位链路，请避免引入业务框架、项目私有配置或生产运行依赖。

## 开发环境

- Node.js >= 18（工具运行时仍兼容 Node.js >= 14）
- macOS 或 Windows（平台相关功能需要在对应系统上验证）

仓库没有第三方运行时依赖，克隆后可以直接运行测试：

```bash
npm test
npm pack --dry-run
```

## 修改原则

- 保持生产构建零注入、零服务、零运行时痕迹。
- 点击定位只能旁观业务 tap，不能吞掉、代理或重放业务事件。
- 新增构建适配时优先复用 `transformTemplate` 和现有运行时协议。
- 涉及 macOS、Windows、webpack4 或 webpack5 的修改，应说明实际验证范围。
- 新行为必须补充聚焦测试，修复回归时先增加可复现用例。

## 提交 Pull Request

Pull Request 请说明问题、实现方式、影响范围和验证命令。不要提交构建产物、真实业务页面、账号信息、密钥或本机绝对路径。
